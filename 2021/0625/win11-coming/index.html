Windows 11 问世了！

北京时间6月24日晚间，微软公司面向全球正式发布了旗下 Windows 操作系统的最新版本 ——Windows 11。

看点一:

全新设计的UI

Windows 11 最让人眼前一亮的变化，是它在 UI 设计上的重大更新。

首先从开始菜单来看，Windows 11 的“开始”菜单删除了最初随 Windows 8 引入的动态磁贴，增添了用户在其他设备的常用启动应用。

在位置方面，Windows 11 的开始菜单也进行了重新布局设计 —— 首次从左侧转移到中间。

同时，在云和 Microsoft 365 的加持下，“开始”菜单能够显示最近文档，无论用户在哪个平台或设备查看过，即便是在 Android 或 iOS 设备上，都能够在“开始”菜单上查看到浏览记录。

在多任务处理上，Windows 11 推出了全新的贴靠布局、贴靠群组和虚拟桌面功能，能够帮助用户整理窗口和优化屏幕空间。

比如，在虚拟桌面功能上，用户可以根据生活和工作单独创建虚拟桌面，同时还能根据个人喜好进行个性化定制。

其它方面，Windows 11 还支持明亮模式和暗黑模式。

总的来看，本次 Windows 11 在 UI 上带来了许多新设计，但 Windows 11 大部分 UI 设计与 Windows 10X 十分类似，后者是针对双屏设备计划推出的项目，但最终被微软取消。

另外，在界面居中设计和部分图标、窗口的圆角设计也与苹果 macOS 有着异曲同工之处，不得不说，全新的 Windows 11，有 macOS 内味儿。

看点二:

升级 PC 游戏体验

除了界面的更新变化，Windows 11 也带来了性能上的提升。

在游戏体验上，Windows 11 从画面体验、性能提升，以及外设支持等多个方面带来了改变，具体表现为：

DirectX 12 Ultimate 可以在高帧率下提供沉浸式画面体验；

DirectStorage（直通存储）能够减少加载时间，同时提升游戏体验；

Auto HDR 能够提供更加宽广的色域和更丰富的色彩

Windows 11 支持大部分 PC 游戏配件和外设，用户可以通过 Xbox Game Pass for PC 或 Xbox Game Pass Ultimate 访问 PC 游戏。

新增窗口插件 Wigets

在 Windows 11 上，微软带来回归了全新的 Widgets 功能。与此前小部件不同的是，Widgets 更加贴合全新 Windows 系统的设计。

据微软方面介绍，Widgets 由 Microsoft Edge 浏览器驱动，是一项个性化提示功能。

用户可以直接在电脑桌面上开启一个管理视图，接收新闻动态、天气变化或者消息通知。当用户打开个性化提示框后，Wigets 支持在屏幕上滑动，从而不对界面上的其它工作造成影响。

全新的 Microsoft Store

在 Windows 11 中，微软为用户打造了一个全新的 Microsoft Store （微软商店）。

简单来说，Microsoft Store 其实就是微软试图向苹果 App Store 学习而打造一个同类产品，用户可以在此探索应用程序和观看、创作、游戏、工作及学习相关的内容。

看点三:

Windows 11 可以本地运行 Android App

Windows 11 的另外一个大亮点是：它可以原生运行 Android 应用。

这在 Windows 发展史上，是第一次。

在发布会现场，微软演示了如何通过 Microsoft Store 中找到 Android 应用程序，然后通过亚马逊的应用商店进行下载 —— 值得一提的是，在发布会演示中，微软选择了字节跳动旗下的 TikTok App 来演示这一功能。

值得注意的是，这个功能可能需要在今年晚些时候上线。

为了在 Windows 11 可以本地运行 Android App，微软与 Intel 和亚马逊都进行了深度合作，其中在技术层面合作更加深入的是 Intel—— 具体来说，Windows 11 对 Android App 的支持，得益于 Intel Bridge 技术。

据了解，Intel Bridge 技术是一种运行时后期编译器，它能够让移动应用在基于 x86 的设备上以“原生应用”形态运行，包括在 Windows 上运行。通过将 CPU 核心、图形显卡技术、人工智能加速器、图形处理器等组件集成到单个经过验证的解决方案中，Intel 的多架构 XPU 策略可为合适的工作负载提供合适的引擎。

某种程度上，Intel Bridge 技术更像是 Windows 11 专门定制的，它的确架起了 Windows 应用生态和 Android 生态的桥梁。

不知道 Google 看到 Windows 11 的这个功能，会作何反应。

开发者支持

当然，基于 Windows 11 的到来，微软也在应用生态的构建上做出努力。

在发布会上，微软表示正在采取措施进一步开放 Microsoft Store，为创作者和开发者提供更多盈利的机会。

比如说，微软让开发者和独立软件开发商（ISVs）推出他们的应用程序，无论是 Win32 应用、Progressive Web App（PWA）、通用 Windows 应用程序（UWP）或任何其他应用框架（比如说 Android），微软都可以在 Microsoft Store 中进行接纳。

同时，微软还宣布对收入分成政策进行渐进式改革，应用开发者可以将他们自己的商贸平台引入到 Microsoft Store，并保留 100% 的收入，微软不收取任何费用（不包含游戏）。

另外，应用开发人员仍可沿用微软的 85/15 这一具有竞争力的收益分成模式。

总体来看，微软试图在本次更新中体现出更为开放的心态，其目的自然是为了吸引开发者，最终促进自家 Windows 应用生态的不断扩大。
